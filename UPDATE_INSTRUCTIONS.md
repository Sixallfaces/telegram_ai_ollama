# Как обновлять репозиторий (Mac + GitHub Desktop)

Этот файл объясняет, как быстро получать последние изменения из репозитория после того, как я внесу правки в ветку `main`.

## Вариант A — Терминал (рекомендуется)

1. Откройте Terminal и перейдите в папку проекта:
   ```bash
   cd /путь/к/проекту
   ```
2. Выполните один раз (если ещё не сделали), чтобы сделать скрипт исполняемым:
   ```bash
   chmod +x update.sh
   ```
3. Каждый раз, чтобы получить последние изменения, запустите:
   ```bash
   ./update.sh
   ```

Скрипт `update.sh` делает безопасный `fetch` и `reset --hard` к `origin/main`, но предварительно проверяет, что рабочее дерево чистое (нет незакоммиченных изменений).

Если вы хотите принудительно перезаписать локальные изменения, выполните (внимание: это удалит незакоммиченные изменения):
```bash
git fetch origin && git reset --hard origin/main
```

## Вариант B — GitHub Desktop (GUI)

1. Откройте GitHub Desktop и выберите репозиторий `Sixallfaces/telegram_ai_ollama`.
2. Убедитесь, что выбрана ветка `main`.
3. Нажмите `Fetch origin`. Если есть изменения — нажмите `Pull origin`.
4. Если у вас есть незакоммиченные изменения, GitHub Desktop предложит закоммитить или стешировать — выполните одно из действий перед pull.

## Полезные команды

- Посмотреть статус:
  ```bash
  git status
  ```
- Сделать обычный безопасный pull:
  ```bash
  git checkout main && git pull origin main
  ```
- Принудительно синхронизироваться (удаляет локальные изменения):
  ```bash
  git fetch origin && git reset --hard origin/main
  ```
- Проверить текущий коммит:
  ```bash
  git rev-parse --short HEAD
  ```

## Работа с незакоммиченными изменениями

- Сохранить изменения временно (stash):
  ```bash
  git stash save "WIP"
  git stash pop  # вернуть после обновления
  ```
- Или закоммитить локально:
  ```bash
  git add . && git commit -m "WIP: local changes"
  ```

- Скрипт `update.sh` намеренно откажется работать при наличии незакоммиченных изменений — это защищает от случайн��й потери данных.
